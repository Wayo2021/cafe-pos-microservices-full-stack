├── controller/    (ชั้นรับแขก) 
 │                  รับ Request จากหน้าบ้าน (หรือ Gateway) และส่ง Response (JSON) กลับไป 
 │                  *เปรียบเสมือนพนักงานรับออเดอร์หน้าร้าน*
 │
 ├── service/       (ชั้นประมวลผล / Business Logic)
 │                  คิดเลข, ตรวจสอบเงื่อนไข, ดึงข้อมูลจากหลายๆ ที่มาประกอบร่างกัน
 │                  *เปรียบเสมือนพ่อครัวที่ทำอาหารตามออเดอร์*
 │
 ├── repository/    (ชั้นคลังสินค้า)
 │                  มีหน้าที่คุยกับ Database (PostgreSQL) อย่างเดียว (Save, Find, Delete)
 │                  *เปรียบเสมือนพนักงานคลังสินค้าที่ไปหยิบของตามที่พ่อครัวสั่ง*
 │
 ├── entity/        (พิมพ์เขียวฐานข้อมูล)
 │                  ไฟล์ Java ที่ตัวมันเองคือ "ตาราง" ใน Database (ที่เราสร้าง Product.java ไปแล้ว)
 │
 ├── dto/           (Data Transfer Object - กล่องพัสดุ)
 │                  เอาไว้กำหนดหน้าตาข้อมูล (JSON) ที่เราจะรับเข้ามาหรือส่งออกไป โดยไม่ต้องเผยแพร่ Entity ตรงๆ
 │
 ├── exception/     (แผนกจัดการปัญหา)
 │                  เอาไว้ดักจับ Error เช่น หาเมนูน้ำไม่เจอ (ProductNotFound) แล้วตอบกลับเป็น JSON สวยๆ
 │
 └── config/        (ตั้งค่าระบบ)
                    เช่น การตั้งค่า Security, การตั้งค่าอนุญาตให้หน้าบ้านข้ามโดเมนมาเรียกได้ (CORS) ##
